## 1. 目的・背景

ビルオーナー／管理会社／テナント／仲介会社など関係者間の煩雑な業務を **Web＋クラウド上** に集約し、

手間・コスト削減と収益最大化を図ることが目的です。主サービスは以下の 7 カテゴリです。

| # | サービスカテゴリ     | コア価値                       |
| - | -------------------- | ------------------------------ |
| 1 | ビル経営全般サポート | 効率化・手間削減               |
| 2 | テナントリーシング   | 空室時に 1 クリックで情報発信 |
| 3 | テナント対応         | 各種やり取りを HP 上で完結     |
| 4 | ビルメンテナンス     | 分離発注の一元管理・修繕支援   |
| 5 | トラブル対応         | 緊急時の業者/弁護士紹介        |
| 6 | 資金調達             | CF・部分売却・借地化 等        |
| 7 | 収入アップ支援       | 清掃/工事/会議室運営 etc.      |

## 2. ユーザロール・権限

ログイン入口は 4 ロール。

| ロールID | ロール名     | 典型ユーザ                   | 主な権限             |
| -------- | ------------ | ---------------------------- | -------------------- |
| OWNER    | ビルオーナー | 不動産オーナー・資産管理会社 | 全機能閲覧・承認     |
| MGMT     | 管理会社     | ビル管理委託会社             | 日常管理・請求処理   |
| TENANT   | テナント     | 入居企業                     | 予約・申請・緊急連絡 |
| BROKER   | 仲介会社     | 客付け会社                   | 空室情報閲覧・資料DL |

---

## 3. 機能要件（ロール別）

### 3.1 オーナー（OWNER）

* **ダッシュボード**
  * イベントカレンダー、TODO、緊急連絡一覧、最新インフォメーション表示
* **受付・承認** （ワークフロー）
* 予約・申請の承認／却下（例: 会議室予約、工事申請）
* **管理状況確認**
  * テナント管理、会議室管理、メンテナンス管理、空室管理、工事管理等のモジュール化一覧
* **緊急連絡管理**
  * 緊急通報閲覧・対応依頼ステータス管理

### 3.2 管理会社（MGMT）

* OWNER 機能＋請求書発行、業務報告、イベント管理など専用タブ

### 3.3 テナント（TENANT）

* イベントカレンダー、手続き申請、館内細則閲覧、緊急連絡、インフォメーション

### 3.4 仲介会社（BROKER）

* 空室情報確認、図面DL、契約書・館内細則DL、インフォメーション
