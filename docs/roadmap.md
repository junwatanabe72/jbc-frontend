開発ロードマップ（フロントエンド）

  フェーズ1: 基盤構築 🔥 最優先

  期間: 2-3週間
  対象画面: LGN-001, OWN-DASH, TEN-DASH の基本版

1. 技術スタック選定
   - React + TypeScript + Vite
   - Tailwind CSS (スタイリング)
   - React Router (ルーティング)
   - Zustand/Context API (状態管理)
2. 認証システム
   - ログイン画面 (LGN-001)
   - ロール別ルーティング (OWNER/MGMT/TENANT/BROKER)
   - Protected Routes実装
3. ダッシュボード基盤
   - オーナーダッシュボード (OWN-DASH) - 基本レイアウト
   - テナントダッシュボード (TEN-DASH) - 基本レイアウト
   - 共通コンポーネント (Header, Sidebar, Layout)

  フェーズ2: カレンダー・イベント機能 📅

  期間: 2-3週間対象画面: OWN-CAL, イベント詳細画面

1. カレンダーコンポーネント
   - 月/週/日表示切り替え
   - イベント表示・作成・編集・削除
   - API連携 (GET/POST /events)
2. イベント管理
   - イベント詳細モーダル
   - 繰り返し予定対応
   - 通知機能基盤

  フェーズ3: 申請・承認ワークフロー ✅

  期間: 2-3週間
  対象画面: OWN-REQ, 申請一覧・詳細

1. 申請管理画面
   - 申請一覧表示
   - 承認/却下ボタン
   - ステータス管理
2. 申請フォーム
   - 動的フォーム生成
   - ファイルアップロード
   - 承認フロー可視化

  フェーズ4: 予約・管理機能 🏢

  期間: 3-4週間
  対象画面: 会議室予約、管理状況画面

1. 予約システム
   - 会議室空き状況表示
   - 予約作成・変更・キャンセル
   - 予約カレンダー連携
2. 管理機能モジュール
   - テナント管理
   - メンテナンス管理
   - 空室管理
   - 工事管理

  推奨実装順序

⏺ Update Todos
  ⎿  ☒ doc.mdを分析してフロントエンド実装の優先順位を決定
     ☐ 技術スタック選定とプロジェクト構造設計
     ☐ フェーズ1: 認証とダッシュボード基盤の実装
     ☐ フェーズ2: カレンダー・イベント管理機能
     ☐ フェーズ3: 申請・承認ワークフロー
     ☐ フェーズ4: 予約システムと管理機能

⏺ 1. 認証システム → 全ての機能の前提
  2. ダッシュボード基盤 → ユーザーが最初に見る画面
  3. カレンダー機能 → 最も使用頻度が高い
  4. 申請・承認機能 → ビジネス価値が高い
  5. 予約システム → 複雑だが重要
  6. 管理機能 → 最後に実装

  技術選定理由

- React + TypeScript: ロール別の複雑な権限管理に最適
- Tailwind CSS: レスポンシブデザインとコンポーネント再利用性
- Vite: 高速な開発環境
- Zustand: 軽量でシンプルな状態管理

  この順序で実装すれば、早期に動作するプロトタイプを作成でき、段階的に機能を追加できます。
  フェーズ1から始めることをお勧めします。
